What is akjon.dev?

Are you a developer looking for an awesome name for your personal website or projects? akjon.dev gives you a free subdomain that points to your static site via a CNAME DNS record.

The entire registry lives in a public GitHub repository. You claim your subdomain by submitting a pull request. Works with GitHub Pages, Netlify, Vercel, and any other static host that supports custom domains.

This service includes full registration across 3 distinct domain spaces:

yourname.akjon.dev yourname.code-space.me yourname.id-card.me

How to claim your subdomain

Four steps, all done through GitHub.

01

Star and fork

Star the repository, then fork it to your own GitHub account.

02

Set up your site

Deploy your site to GitHub Pages, Netlify, Vercel, or any static host. Get the CNAME value (e.g. yourname.github.io).

03

Add your subdomain file

Create your file inside the specific target directory for the domain extension you choose inside the domains/ folder layout.

04

Submit a pull request

Open a PR against the main repo. Once merged, your global custom subdomain record will instantly go live.

Repository Layout Structure

Place your configuration file directly in the directory matching your chosen domain suffix:

domains/
├── akjon.dev/
│   └── sabbir.json
├── code-space.me/
│   └── sabbir.json
└── id-card.me/
    └── joshim.json

The Subdomain Payload Template

Configure your entry exactly using this schema layout format structure:

example.json JSON 
{
  "owner": {
    "username": "exampleuser",
    "email": "hello@example.com"
  },
  "records": {
    "CNAME": "exampleuser.github.io"
  }
}

Replace placeholder keys with your respective active details. CNAME properties direct easily to targets matching GitHub Pages, Netlify, or Vercel endpoints.

Ready to claim yours?

Fork the repo, add your entry inside the target directory, and open a pull request. Your subdomain goes live once it's merged.
